# 智能巡检小车项目说明

## 当前代码状态

- 主控为 `STM32F407ZGT6`，工程使用标准外设库，不是 CubeMX/HAL 工程。
- 当前真正接入运行的是 `App/main.c` 的任务选择调度，以及 `App/task1.c` 的任务1主循环。
- 任务2、任务3、任务4目前只有占位函数，尚未开始实现。
- OpenMV 现在按“常在线、主动识别、主动上报”方式工作，STM32 只接收识别结果，不再发送触发命令。
- 语音模块全程在线，摄像头全程在线。

## 当前运行流程

### 1. 上电

- 初始化 PWM、电机、编码器、TIM6、USART1/2/3、红外、MPU、语音、报警和按键。
- 语音播报“上电完成”。
- 蜂鸣器短响一声。

### 2. 模式选择

- 使用 `PA15` 按键选择任务。
- 短按切换 `任务1 -> 任务2 -> 任务3 -> 任务4 -> 任务1`。
- 长按进入当前任务。

### 3. 任务现状

- 任务1：已接入调度，代码在 `App/task1.c`。
- 任务2：未实现。
- 任务3：未实现。
- 任务4：未实现。

## 任务1现状

- 当前任务1使用 `App/task1.c` 中的独立循环。
- 进入任务1后默认从 A 点开始。
- 直线路段用编码器累计脉冲做区间判断。
- 转弯路段用 MPU 偏航角做阶段判断。
- 巡线仍由 5 路红外 + PID 速度环完成。

说明：

- 任务1这套逻辑已经写入代码，但仍需要结合实车继续校准脉冲阈值、方向符号和转弯判定。
- 任务2/3/4当前不要按“已完成”理解，README 也不再写成已实现。

## OpenMV 通信说明

### 当前模式

- OpenMV 常在线运行 `OpenMV/main.py`。
- OpenMV 自己持续识别。
- 识别到目标后主动通过串口上报 8 字节结果帧。
- STM32 侧只解析接收到的结果，不再向 OpenMV 发送检测触发命令。

### 当前有效帧格式

OpenMV -> STM32:

| 字节 | 内容 | 说明 |
|------|------|------|
| 0 | `0xBB` | 帧头 |
| 1 | `ID` | 物体ID |
| 2 | `Xh` | X坐标高字节 |
| 3 | `Xl` | X坐标低字节 |
| 4 | `Yh` | Y坐标高字节 |
| 5 | `Yl` | Y坐标低字节 |
| 6 | `CHK` | 校验和 |
| 7 | `0x55` | 帧尾 |

### 物体ID

| ID | 物体 |
|----|------|
| `0x01` | 打火机 |
| `0x02` | 剪刀 |
| `0x03` | 锤子 |

## 主要硬件引脚

| 功能 | 引脚 | 说明 |
|------|------|------|
| 电机 PWM | `PA8~PA11` | TIM1 四路 PWM |
| 电机方向 | `PC0~PC7` | 四个电机方向控制 |
| 左编码器 | `PA0, PA1` | TIM2 |
| 右编码器 | `PA6, PA7` | TIM3 |
| OpenMV 串口 | `PB6, PB7` | USART1 |
| 语音模块串口 | `PA2, PA3` | USART2 |
| 姿态模块串口 | `PB10, PB11` | USART3 |
| 红外巡线 | `PD1~PD5` | 5路输入 |
| 启动按键 | `PA15` | 上拉输入 |

## 当前代码里的几个关键事实

- `App/main.c` 里仍然是按键选任务，不是上电自动直接跑任务。
- `App/state_machine.c` 只是保留框架，目前不是主执行入口。
- `App/openmv.c` 已改为接收优先，发送触发链路已断开。
- `App/task1.c` 中同时放了 `Task2_Run/Task3_Run/Task4_Run` 的占位实现，这三项后续需要单独展开。
